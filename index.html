Project Overview

AAIG Portal

A portal for ID card generation and automated attendance management system

An automated attendance system and ID card generator that consists of a web system for entire organization to record attendance and generate identity cards of students and staff. The web application is to be used by students, faculty, and admin personnel.
  • Role of a faculty:
    1. Each Faculty has his/her own login. On login, the class faculty can see a list of students or members registered under it.
    2. The faculty may take the attendance and mark the present stu- dents using checkbox provided in front of every student name.
    3. This attendance sheet is stored and sent to the central adminis- trator of the organization and stored there.
    4. Faculty can also view his/her identity card if generated.
  • Role of an admin:
    1. The role of the admin is to add new student and faculty by entering his personal details and provides the faculty/student with userID and password so that he/she can access the application.
    2. Admin can check the attendance of the particular student by entering the roll number and generate attendance reports.
    3. The admin may check all attendance data, press a button to see defaulter list, search for particular student attendance by name, search class attendance and generate excel reports.
    4. Admin can easily generate identity cards of students and employ- ees by simply entering their details which is required to be dis- played in the ID card in text fields. This can then be given for printing.
  • Role of a Student:
    1. The student logs in and checks his/her attendance statistics for different courses/faculties.
    2. The student also has the option to register for course available and once he registers for the course, his name will appear for attendance whenever a faculty logs in to record the attendance.
    3. Students can also view their identity cards if generated.
